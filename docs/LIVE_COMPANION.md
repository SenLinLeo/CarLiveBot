# 与火山引擎直播伴侣对接步骤

CarLiveBot 负责生成**文本与语音流**，直播伴侣负责将数字人音视频推流到抖音。两者对接方式如下。

## 1. 直播伴侣端

1. 从**火山引擎企业直播控制台**下载并安装直播伴侣（仅支持 Windows）。
2. 使用已开通企业直播的火山引擎账号登录。
3. **设置 → 推流**：推流模式 RTMP，超时 30 秒，开启「自动重连」。

## 2. 音视频源来源

- **方式 A**：数字人渲染由第三方/本地进程完成，接收 CarLiveBot 的**文本或音频**，输出带唇形同步的音视频流；直播伴侣添加「媒体源」，填入该流地址（如 `rtmp://localhost:1935/live`）。
- **方式 B**：若 CarLiveBot 后续扩展为提供 RTMP 或音频管道，直播伴侣添加媒体源指向该地址即可。

当前版本 CarLiveBot 提供：
- **管理 API**：`POST /api/v1/script/generate` 获取文案；若需音频，可对接应用层将 TTS 流写入文件或管道供直播伴侣/数字人使用。
- **合规文案**：`GET /api/v1/compliance/label` 返回「本直播由AI虚拟人提供服务」等标注，用于在直播伴侣中配置**文字挂件**。

## 3. 合规标注（必做，避免封号）

1. 在直播伴侣「场景编辑」中点击「挂件」→「文字挂件」。
2. 内容填写：**本直播由AI虚拟人提供服务**（或与 `configs/config.yaml` 中 `compliance.label_text` 一致）。
3. 位置：直播间左下角或右下角，字体 ≥ 20，颜色与背景对比明显，**全程可见**。

## 4. 抖音开播

1. 直播伴侣「直播设置」→「转推」→ 添加平台「抖音」，使用已开直播权限的抖音账号扫码授权。
2. 获取推流地址与推流码后，开启推流开关。
3. 建议码率 4000kbps（720P）、帧率 30fps、关键帧间隔 2 秒。

## 5. 日常运维

- 开播前：确认 CarLiveBot 服务与配置可用（`/health`、门店配置）。
- 直播中：关注直播伴侣推流状态与延迟（建议 ≤ 3 秒）。
- 断流：依赖直播伴侣「自动重连」；CarLiveBot 侧可配合监控与告警（见配置中的 `ALERT_WEBHOOK_URL` 等）。
